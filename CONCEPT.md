# AutoVPN — technical overview

## Одна строка

Приложение для доступа к ограниченному контенту: нажал кнопку — всё работает. Бесплатно, без регистрации, без настроек.

---

## Проблема

Часть интернет-сервисов ограничена по географическому принципу. Пользователь хочет полный доступ, но не хочет разбираться как.

Существующие решения:
1. **Коммерческие VPN** (NordVPN, ExpressVPN) — платная подписка, регистрация, часто сами блокируются
2. **Бесплатные VPN из маркетов** — реклама, слежка, ненадёжные
3. **Технические инструменты** (v2rayNG, NekoBox) — найди сервер, установи клиент, импортируй конфиг, пойми что сервер сдох, повтори. Это делают технари. Обычный пользователь — нет

Все три варианта требуют от пользователя разбираться в том, что ему не нужно знать.

## Решение

Одно приложение, одна кнопка. Всё остальное — автоматически:

```
[ CONNECT ]
     ↓
Скачать конфиги → Запустить sing-box → Подключено
     ↓                    ↓
  (каждые 2ч)       sing-box сам:
                    - тестирует серверы (urltest)
                    - выбирает лучший по latency
                    - переключает при отказе
                    - kill switch (strict_route)
```

AutoVPN — **агрегатор**: собирает серверы из открытых источников, тестирует, подключает к лучшему. Своих серверов нет. Пользователь не знает и не должен знать что происходит под капотом.

**Ценностное предложение:**
- Полный доступ к интернету — одна кнопка
- Бесплатно — нет подписки, нет платежей
- Без регистрации — нет email, нет аккаунта, нет данных
- Автоматически — серверы, failover, обновления

---

## Целевая аудитория

Люди, которые хотят полный доступ к интернету, но не могут или не хотят настраивать технические VPN-инструменты. Без фильтра по возрасту, профессии или навыкам.

## Как работает

### Конфиги

Приложение агрегирует VLESS-серверы из открытых репозиториев на GitHub:

| Источник | Что берём | Роль |
|---|---|---|
| [vpn-configs-for-russia](https://github.com/igareck/vpn-configs-for-russia) | VLESS+Reality серверы (20-50 шт), обновляются каждые 1-2 часа | Основной |
| [F0rc3Run](https://github.com/F0rc3Run/F0rc3Run) | Только `security=reality` (~90 шт из 500+), фильтруем остальное | Дополнительный |

Источники — чужие публичные репозитории. Мы полностью полагаемся на них и не планируем менять пока работают. Если источник пропадёт — заменяем на другой.

Приложение скачивает конфиги, дедуплицирует, мержит. Если GitHub недоступен — фолбэк на CDN-зеркала (jsDelivr, Statically), затем на локальный кэш.

> **Планируется:** вшивать набор конфигов в бинарник при сборке — для случаев когда нет ни сети, ни кэша (первый запуск при полной блокировке).

### Тестирование, failover, выбор сервера
Делает **sing-box нативно** через `urltest` outbound group:
- Автоматически тестирует все серверы каждые 3 минуты
- Выбирает с наименьшим latency
- При отказе сервера — мгновенно переключается на следующий
- Наше приложение только генерирует конфиг и запускает sing-box

### Kill switch
`strict_route: true` в конфиге tun — весь трафик устройства идёт в туннель на уровне ОС. Даже при падении sing-box правила маршрутизации остаются.

### Мониторинг
sing-box поднимает Clash API (`127.0.0.1:9090`). Приложение опрашивает его для UI: текущий сервер, latency, список серверов и их статус.

## Архитектура

Приложение написано на **Go** — тот же язык что и sing-box. Это позволяет использовать sing-box как библиотеку напрямую, без управления отдельным процессом.

```
┌──────────────────────────────────────────────┐
│              UI (Wails)                      │
│     Go backend + HTML/CSS/JS frontend        │
├──────────────────────────────────────────────┤
│            autovpn-core (Go)                 │
│  ConfigFetcher, VlessParser,                 │
│  SingBoxConfigBuilder, ClashApiClient        │
├──────────────────────────────────────────────┤
│         sing-box (Go library)                │
│  urltest, selector, tun, clash_api           │
└──────────────────────────────────────────────┘
```

**Десктоп (Linux, macOS, Windows):**
- Один Go-бинарник, sing-box встроен как библиотека
- UI через Wails (Go + веб-фронтенд)
- tun создаётся sing-box напрямую
- Zero external dependencies

**Мобильные (Android, iOS):**
- Go-ядро компилируется через gomobile в .aar / .xcframework
- Тонкая нативная обёртка:
  - Android: Kotlin VpnService + autovpn-core.aar
  - iOS: Swift NetworkExtension + autovpn-core.xcframework
- Тот же подход что у sing-box-for-android / sing-box-for-apple

**Преимущества Go:**
- sing-box — Go библиотека, импортируется напрямую, без IPC
- Нативный бинарник, zero runtime (~10-15MB вместо ~100MB с JVM)
- gomobile для мобильных платформ — проверенный путь (sing-box так и делает)
- Один язык для core + desktop app

## UX

### Первый запуск — флоу нового пользователя

Никаких визардов, регистраций, настроек. Пользователь видит одну кнопку и нажимает её.

**Шаг 1: Открыл приложение**
```
┌───────────────────────────────┐
│                               │
│         ◉  AutoVPN           │
│      [   CONNECT   ]         │
│                               │
│                               │
└───────────────────────────────┘
```

**Шаг 2: Нажал Connect — запрос прав (один раз, показывает ОС)**
- Android: "AutoVPN wants to set up a VPN connection" → OK
- iOS: "AutoVPN Would Like to Add VPN Configurations" → Allow → Face ID
- Desktop: polkit / UAC / системный пароль для создания туннеля
- Android 13+: дополнительно запрос на уведомления (нужны для foreground service)

**Шаг 3: Подключение**
```
┌───────────────────────────────┐
│                               │
│         ◉  AutoVPN           │
│     [ ··· CONNECTING ]        │
│                               │
│    Downloading configs...     │
│                               │
└───────────────────────────────┘
```
Под капотом: fetch конфигов → генерация sing-box JSON → запуск sing-box → urltest тестирует серверы.

**Шаг 4: Подключено**
```
┌───────────────────────────────┐
│                               │
│         ◉  AutoVPN           │
│      [  DISCONNECT  ]        │
│                               │
│    Frankfurt  42ms            │
│    5 / 23 servers available   │
│                               │
└───────────────────────────────┘
```

Весь флоу: **открыл → нажал → подтвердил → подключено**. 10 секунд.

### Если интернет заблокирован при первом запуске

Приложение автоматически пробует фолбэк-цепочку:
1. GitHub → CDN-зеркала → локальный кэш
2. *(планируется)* Конфиги, вшитые в бинарник при сборке — для первого запуска без сети и кэша

### Повседневное использование

- Мобильные: foreground-уведомление с кнопкой отключения
- Десктоп: иконка в системном трее
- Конфиги обновляются в фоне каждые 2 часа
- Серверы тестируются sing-box каждые 3 минуты
- При отказе сервера — мгновенное переключение, пользователь не замечает

## Ограничения

- **Серверы — третьих сторон.** AutoVPN не контролирует серверы. Скорость и доступность зависят от того что есть в открытых источниках
- **Нет SLA.** Бесплатный продукт без гарантий стабильности
- **Приватность трафика.** Приложение не собирает данных, но трафик проходит через чужие серверы — их операторы теоретически могут его видеть. HTTPS защищает содержимое, но не метаданные
- **Зависимость от источников.** Если все GitHub-репозитории с конфигами исчезнут — приложение перестанет находить серверы
